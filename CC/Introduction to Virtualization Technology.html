    Virtual Machine (Pseudo Machine)

    • Expose only one “bare” machine interface

    • Support only one instance of an operating system kernel

    • It is an environment created by a Virtual Machine Monitor

      Virtual Machine Monitor (VMM)

    • Sits between the bare hardware and the OS

    • Each virtual machine interfaces with its host system via the VMM

    • To work with the VMM to coordinate access to the physical host system's hardware resources

    • Lightweight operating shell - to provide VM hosting services

    • Host : System which accesses the “real” hardware

    • Guest : System which accesses the “virtual” hardware

    Approaches

    Instruction Set Architecture (ISA) Level

    • Emulating an instruction set architecture completely in software

    • To execute instructions issued by the guest machine by translating them to a set of native instructions and then executing them on the available hardware.

      Crusoe


    • Dynamic x86 emulator – “code morphing engine”

    • To execute any x86 based application on top

    • Simpler, Smaller, and Less power consuming chip

    • To use 16MB system memory as “translation cache”

    • To store recent results of the x86 to VLIW instruction translations

    • Shadow registers: Enables easy exception handling 21

      Introduction to Virtualization Technology


      Approaches

      Bochs


    • Open-source x86 PC emulator - Highly portable

    • C++ by Kevil Lawton

    • Platforms: x86, PowerPC, Alpha, Sun, and MIPS

    • Emulate : x86 machines including 386, 486,Pentium, Pentium Pro or AMD64 CPU, including optional MMX, SSE, SSE2, and 3D

    • To interpret every instruction from power-up to reboot, emulates the Intel x86 CPU, a custom BIOS, and has device models for all of the standard PC peripherals

    • Writing a new device driver, understanding how the hardware devices work and

      interact with others

    • To support legacy applications on modern hardware, and as a reference model when testing new x86-compatible hardware. 22

      Introduction to Virtualization Technology


      Approaches

      QEMU


    • Fast processor emulator - Portable dynamic translator

    • User space mode

      • Linux processes compiled for one CPU on another CPU

      • For cross-compilation and cross-debugging

    • Full system emulation mode

      • a processor and several peripheral devices

      • Multiple target ISAs: x86, ARM, PowerPC, Sparc

      • Supports self-modifying code

      • Full-software and simulated MMU

    • Dynamic translation

    • To use a 16MB translation cache and flushes to empty when it gets filled

            Operating System (OS) Level


  • To partitions the physical machine’s resources at the operating system level

  • All the OS-level VMs share a single operating system kernel

  • VM can share the execution environment of the host machine, but any state

    changes in the VM are conﬁned in the VM’s local environment

    Jail

  • To provide the ability to partition an operating system environment

  • To capture system resources and data structures such as processes, file system, network resources, etc.

  • When the system is booted up after a fresh install, no processes will be in jail.

  • When a process is placed in a jail, all of its descendants after the jail creation,

    along with itself, remain within the jail.

    Approaches

    WINE

  • Windows Emulator - Wine Is Not an Emulator

    Wine Server NT-Kernel

  • Wine is not a virtual machine, it does not emulate a processor, and one is not supposed to install an operating system or device drivers on top of it.

  • Wine is an implementation of the Windows API, and can be used as a library to port Windows applications to Unix.

  • Wine does look like Windows to the Windows binaries (.EXE files) and emulates its behavior very closely to that of Windows.

  • Wine is a virtualization layer on top of X and

Unix to export the Windows API/ABI, thus

letting the Windows binaries run on top of it. 34
